XNet is a step forward in filesharing

XNet offers an alternative to focusing on one-to-one downloads, which are simply slow. It's also different from torrents, which use excessive amounts of bandwidth. XNet calculates the best implementation of chunking.

Ease of Use

XNet is equipped with a gorgeous skin with a GUI layout. It's just buttons, so set it and forget it.

Speed

XNet utilizes standard peer to peer networking so you can have multiple file sources. It also uses a block size of 512kb, which strikes a balance between reliability and bandwidth use.

Open Source

XNet's internals are out for everyone to see. Look at the bottom of this page for a link to its GitHub repo.

When will XNet be formally released?

XNet is currently in alpha/beta testing. It is predicted to be released as a candidate around 3/14/15. You can build it yourself from source to play around with it, but no support will be provided -- nor will bootstrapping.
